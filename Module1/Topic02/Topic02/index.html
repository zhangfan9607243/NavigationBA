    Topic 2 - Python 程序基本要素¶

    1. 一些简单的 Python 程序¶

    1.1 单行 Python 程序¶

    首先我们来复习一下上一章的打印 Hello World! 的简单程序

    • 我们先在 PyCharm 中打开之前在桌面创建好的项目 PythonBasics
    • 之后在 PythonBasics 总目录下新建一个文件夹，就叫做 Topic02，并在这个文件夹下创建名为 t02_01_single_line.py 的 Python 文件
    • 之后在该文件中写入以下代码并运行：
    print("Hello World!")
    • 输出为：
    Hello World!

    1.2 多行 Python 程序¶

    接着我们来写一个含有多行代码的程序

    • 还是在Topic02文件夹下，我们创建一个名为 t02_02_multi_line.py 的 Python 文件，
    • 之后我们在这个文件中写入以下代码并运行：
    print("Hello World!")
print("Hello Python!")
print("欢迎来到编程世界")
    • 输出为：
    Hello World!
Hello Python!
欢迎来到编程世界

    在 Python 中，代码是按行执行的，并且遵循严格的代码规范：

    • 不可以将两句代码写在一行用空格连接
    • 可以有空白行
    • 除非严格的缩进关系，否则每行代码前面不可以空格
    • 如果一句代码太长，可以使用 \ 连接两行，但是 Python 官方不推荐这样做，因为写出来的代码比较难看

    通过这个简单的多行代码程序，我们可以简单地看出 Python 程序的运行顺序：

    • Python 程序执行时，会将整个文件中的所有代码全部运行，不会跳过任何一行代码，严格按照文件中从上到下的顺序依次运行
    • 如果中途某一行出现错误（Bug），Bug 之后的代码将不会继续运行
    • 如果想让 Python 分步运行，就要用到 PyCharm 中的 debug + 断点运行功能，我们这次不点击运行，而是点击 debug 试试效果
    • 可以看出 debug 这种分步运行方式并不好用，之后我们会介绍在 Python 数据分析中使用一种更好用的分步运行方式 - Jupyter Notebook

    1.3 多文件 Python 程序¶

    目前我们在 Topic 2 文件夹下已经创建了两个 Python 程序：t02_01_single_line.py 与 t02_02_multi_line.py。

    • 在 PyCharm 中，只需右键选择要运行的文件，即可分别运行不同的程序。
    • 如果想让 PyCharm 总是默认运行当前文件，可以在右上角 "configuration" 那里固定到 "current file"

    2. 变量与赋值¶

    在编程中，变量是数据的储存容器，我们可以往容器中放入数据，也可以从容器中取出数据和修改数据，是“避免重复造轮子”最重要的手段之一。

    2.1 变量的定义与赋值¶

    在 Python 中，每个变量在使用前必须先赋值，赋值之后变量才会被创建。

    • 使用等号 = 来给变量赋值：

      • 左边是变量名（容器的名字）
      • 右边是值（要存放的数据）

    • 示例：在 Topic 2 文件夹下已经创建 t02_03_assignment_01.py ，然后定义两个变量 name 和 age ：

    name = "小明"
age = 18

    2.2 变量的使用¶

    定义之后，变量可以直接使用：

    • 我们可以使用 print() 函数查看变量的值
    • 我们也可以使用 type() 函数查看变量的类型，注意在 Python 中，直接使用 type() 函数并不会在控制台输出内容，因此 type() 函数必须嵌套在 print() 函数中实现输出
    • 接着，我们在 t02_03_assignment_01.py 文件中查看以下刚刚定义好的两个变量 name 和 age 的值和类型：
    name = "小明"
age = 18

print(name)
print(age)

print(type(name))
print(type(age))
    • 输出为：
    小明
18
<class 'str'>
<class 'int'>

    在很多编程语言中（例如 Java、C++），存在“常量”的概念 - 定义之后的值不能被修改

    • 但是在 Python 中并没有真正的常量，所有标识符本质上都是变量
    • 唯一能“模拟常量”的方式，就是人为约定：定义一个变量之后不再去修改它的值
    • 我们通常会用全大写字母来表示一个“常量”，例如：PI = 3.14159

    2.3 变量的修改¶

    变量中存储的内容是可以随时改变的

    • 改变一个变量的值，只需把需要改的变量，重新赋予一个新的值即可
    • 例如我们把之前的 age 改为 20：
    age = 18
print(age)

age = 20
print(age)
    • 输出为：
    18
20

    还有一种修改变量数值的方式，那就是增量赋值，增量赋值会在变量原值基础上进行新的运算，新值会覆盖旧值： - 我们来看一下以下例子：

    a = 5
print(a)

a = a + 6
print(a)
    • 输出：
    5
11
    • 更多关于增量赋值的内容我们在下一章再展开来讲

    2.4 变量的删除¶

    在程序运行过程中，变量本质上是保存在内存中的一块空间，用来存储数据，如果某些变量已经不再需要，可以将它们删除，以避免占用内存资源：

    • 在 Python 中，可以使用内置关键字 del 删除变量，变量一旦删除，计算机就不再认识这个变量了，所以调用已经删除的变量就会报错
    • 我们创建一个新的 t02_03_assignment_02.py 文件来演示变量删除：
    age = 18
print(age)

del age 
print(age)
    • 输出为
    18
NameError: name 'age' is not defined

    2.5 变量命名原则¶

    在 Python 中，变量命名包括两个方面 - 命名规则和见名知义的准则，并且这两方面适用于 Python 中所有对象的命名：

    • 变量名
    • 函数名
    • 类名
    • Python文件与路径名

    (1) 命名规则 - 必须遵守¶

    这些规则是 Python 语言强制要求的，如果不遵守会直接报错：

    1. 变量名只能包含大小写字母、数字和下划线，且不能以数字开头

      • ✅：name、age_18、student_name、Dog
      • ❌：my-name（不能有 -）
      • ❌：my name（不能有空格）
      • ❌：123name (以数字开头)

    2. Python 区分大小写：例如 Name、NAME、name 是三个不同的变量

    3. 不能使用 关键字 作为变量名：Python 3.10 版本后一共有35个关键字：
    关键字
    False None True and as
    assert async await break class
    continue def del elif else
    except finally for from global
    if import in is lambda
    nonlocal not or pass raise
    return try while with yield

    (2) 见名知义的准则 - 推荐遵守¶

    这些是良好的编程习惯，能大幅提升代码的可读性和团队协作效率，但是本质来讲以下准则并不是 Python 语言强制的，违反了也并不会导致程序报错：

    1. 有意义的名字：变量名应能体现存储的内容或用途

      • ✅：student_name、price、total_score
      • ❌：name1、data1、tmp
      • ❌：name = 18、age = "张三"

    2. 避免过长或过短：名字要简洁，但不能太含糊

      • ✅：user_id
      • ❌：u
      • ❌：user_identification_code_number_2025

    3. 命名风格一致：Python 推荐以下几种命名方式：

      • 所有字母小写 + 使用下划线连接单词：常用于变量名与函数名，比方说 name、 phone_number、calculate_scores
      • 首字母大写 + 不加下划线连接单词：常用于类名，比方书 Student、BankAccount
      • 所有字母大写 + 使用下划线连接单词：常用与常量名，比方说 PI = 3.1415926

    3. 注释¶

    在编程中，注释 是写在代码里的说明文字，用来帮助程序员理解代码逻辑。注释不会被 Python 解释器执行，只是为了让人更好地阅读代码。

    在 Python 中有两种注释方式，单行注释 和 多行注释

    3.1 单行注释¶

    单行注释以井号 # 开头，井号右边的内容就是注释，可以写在一行的开头，也可以跟在代码的后面。

    • 示例：我们创建一个新的文件 t02_04_comments.py，并运行以下代码：
    # 以下代码展示了 print 函数的使用
print("Hello World!")  # 输出 Hello World
    • 输出：
    Hello World!

    Python 解释器在处理注释时的原理是：

    • Python 解释器在看到井号以后，就知道井号后面是注释了，就自动跳过井号及其后面的内容了。
    • 可以看到，Python 只运行了 print() 函数，自动忽视了那两行的注释

    3.2 多行注释¶

    多行注释适用于文字比较长，使用三个连续的双引号 """注释内容""" 或三个连续的单引号 '''注释内容'''：

    • 示例：我们接着在文件 t02_04_comments.py 中运行以下代码：
    """
以下的代码会：
1. 输出Hello World!
2. 输出Hello Python!
"""

print("Hello World!")
print("Hello Python!")
    • 输出：
    Hello World!
Hello Python!

    使用三个引号的多行注释，与使用井号定义的单行注释，在本质上是不同的：

    • 多行注释的本质是多行字符串，字符串是 Python 中的一种数据类型
    • Python 解释器看到多行注释后，会发现这里有个多行字符串，并从头读到尾
    • 但由于这个多行字符串并没有做任何处理或输出，所以 Python 解释器会将它单纯地摆在这里，不做任何操作
    • 因此，多行字符串可以当做多行注释来使用

    4. 用户输入 - input() 函数¶

    在很多程序中，除了输出信息给用户，还需要接收用户的输入，让程序能够与用户进行交互。

    在 Python 中，获取用户输入要使用内置函数 input()。

    4.1 input() 函数的基本使用¶

    input() 函数的基本语法是： 

    变量名 = input("提示信息：")
    • 首先，input() 函数需要使用一个变量来接收
    • 其次，input() 函数中可以输入一段文字作为提示信息
    • 当 Python 解释器看到有 input() 函数的时候，就会在控制台等待用户输入，用户如果没有输入，程序会一直等待

    • 用户可以输入一段文字在控制台，输入结束后，按回车结束，程序就会继续运行

    • 示例，我们新建一个 t02_05_input.py 文件，并运行以下代码，大家可以尝试输入任意文字，例如我输入了“张三”：

    name = input("请输入你的名字：")
print(name)
    • 输出：
    请输入名字：张三
张三

    4.2 input 返回值的类型¶

    无论用户输入的是数字还是文字，input() 的返回值都是 字符串类型。

    • 示例，假设我们在以下程序中输入“18”：
    age = input("请输入你的年龄：")
print(age)
print(type(age))
    • 输出：
    请输入你的年龄：18
18
<class 'str'>
    • 可以看到，age 变量虽然看起来是数字，但实际是字符串。

    因此，如果想要将用户输入的数字真正变成 Python 中的数字类型，我们就需要转换函数：

    函数 说明
    int(x) 将 x 转换为整数
    float(x) 将 x 转换为浮点数
    • 这次，我们将上面例子中的输入套入一个 int() 函数中，将它转换为 Python 中的整数类型：
    age_new = int(input("请输入你的年龄："))
print(age_new)
print(type(age_new))
    • 输出：
    请输入你的年龄：18
18
<class 'int'>
    • 此外，我们再尝试以下 float() 函数：
    height = float(input("请输入你的身高："))
print(height)
print(type(height))
    • 输出：
    请输入你的身高：178.5
178.5
<class 'float'>
    • 这里我们要注意，int() 函数与 float() 函数中，只能输入“看上去像是数字的文字”，如果输入非数字，就会报错，例如 int("abc") 就会报错。

    5. print() 函数拓展功能¶

    在前面我们已经用过 print() 来输出信息。实际上，print() 函数还有很多拓展功能，可以帮助我们控制输出的格式。

    5.1 print 函数¶

    print() 函数在不加任何的参数的时候，效果是空一行

    • 示例：我们新建一个 t02_06_print_01.py，并且运行以下代码：
    print("Hello World!")
print()
print("Hello Python!")
    • 输出：
    Hello World!

Hello Python!

    5.2 输出多个内容¶

    print() 可以一次输出多个变量或值，使用逗号 , 分隔：

    • 示例：我们在 t02_06_print_01.py 中运行以下代码：
    name = "小明"
age = 18
print("姓名：", name, "年龄：", age)
    • 输出：
    姓名： 小明 年龄： 18
    • 可以看到，Python 默认会在逗号分隔的内容之间加一个空格

    5.2 end 参数 - 控制行尾¶

    默认情况下，print() 在输出内容后会自动换行（相当于在结尾加了一个换行符 \n）。

    • 我们可以通过设置 end 参数，改变行尾的输出内容。

    • 示例：我们新建一个 t02_06_print_02.py，并且运行以下代码：

    print("Hello", end=" ")
print("Python")
    • 输出为：
    Hello Python
    • 如果将 end 中填入空字符串：end=""，就会把前后输出内容连在一起：
    print("Hello", end="")
print("Python")
    • 输出：
    HelloPython

    思考题：如何使用 end 参数输出 Hello Python! 呢？

    • 答案：
    print("Hello", end=" ")
print("Python", end="")
print("!")
    • 输出：
    Hello Python!

    5.3 sep 参数 —— 控制分隔符¶

    在使用 print() 函数进行多个内容输出时，多个输出的内容之间默认用一个空格分隔：

    • 我们可以通过 sep 参数修改分隔符：
    • 示例：我们新建一个 t02_06_print_03.py，并且运行以下代码：
    print("2025", "01", "01", sep="-")
    • 输出：
    2025-01-01
    • 或者把分隔符改成 \ 试一下：
    print("2025", "12", "31", sep="/")
    • 输出：
    2025/12/31

    • 注意，sep 函数只是规定了输出内容两两之间的分隔符，并没有像 end 参数那样，将结尾也换成分隔符

    • 接下来我们看一个 sep 与 end 二合一的综合应用：

    print("2025", "12", "31", sep="-", end=" ")
print("22", "45", "30", sep=":")
    • 结果为：
    2025-12-31 22:45:30

    6. Python 代码规范 (PEP 8)¶

    在团队开发或长期维护中，良好的代码规范比单纯写出能跑的代码更重要。

    Python 官方提供了一份代码风格指南：Python Enhancement Proposal 8 (PEP 8): https://peps.python.org/pep-0008/。
